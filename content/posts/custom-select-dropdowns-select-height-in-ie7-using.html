I’m not a fan of building custom dropdowns, (or any type of custom form field), as they present all types of issues, mainly:

  • Performance (eg, building a dropdown with 200 options)
  • Usability (eg, on mobile devices you want to be able to use the native dropdown control instead of a small custom dropdown box with a scrollbar)
  • Accessibility (it is keyboard accessible, screen reader compliant?)
  • Display (where should the dropdown options be displayed? top, bottom, left, right? - this is obviously dependant on where the <select> element is positioned in the page)

I’m totally against building custom dropdowns for the reasons listed above, but if you find yourself in a situation where you don’t have a choice about it, then I suggest taking the approach of making the <select> element opaque, and sticking a fancy dropdown image behind it. Voila! There’s your fancy looking dropdown.

Advantages:

  • Performance is excellent (it’s a system control)
  • Usability is excellent (the native control is still used on all devices, including most importantly: mobile devices)
  • It’s totally accessible

Disadvantages:

  • You will have a difficult time changing the height of the <select> to match the height of the custom dropdown in IE7. (All other modern browsers will respect the height CSS rule.)

Solution:

I stumbled across this comment the other day on stackoverlow. Basically it’s way to adjust the border on a <select> element by adding a drop shadow, and by adding a drop shadow you’re effectively changing the height of the dropdown. Aha!

After a bit of tinkering, this was the fix I came up with to change the height of a <select> dropdown in IE7: 

select {filter:alpha(opacity=0) progid:DXImageTransform.Microsoft.dropshadow(OffX=0, OffY=10,color=#FFFFFF);}

As the <select> is opaque, the dropdown colour is irrelvant. Tweak the ‘OffY’ value to match the height of the design.
