This is a simple guide on how to get PostgreSQL and phpPgAdmin working on an Ubuntu system.

Install the neccessary packages:

sudo apt-get install postgresql-8.4 postgresql-client-8.4 postgresql-contrib-8.4 phppgadmin

Change the default postgres password and create a new superuser:

sudo -u postgres psql postgres
postgres=# \password postgres
postgres=# \q
sudo -u postgres createuser --superuser $USER
sudo -u postgres psql
postgres=# \password $USER
postgres=# \q

A new phpPgAdmin apache vhost file will symlinked at location:

/etc/apache/conf.d/phppgadmin

You might want to adjust the web server permissions in the original vhost file at location:

/etc/phppgadmin/apache2.conf

If you’ve made any changes to this file you’ll need to restart apache.

sudo apache2ctl configtest
sudo apache2ctl restart

That should be enough to be able to use the user account you previously created to log into phppgadmin. You can access phppgadmin at location ‘hostname/phppgadmin’

Useful commands:

dropdb [databasename]
createdb [databasename]
psql [databasename] < [databasefile].sql
pg_dump [databasename] > [databasefile].sql
